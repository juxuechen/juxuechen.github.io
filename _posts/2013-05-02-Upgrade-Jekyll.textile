<img src="http://fanadir.in/images/jekyll.png" title="jekyll" />

Sudah lama sekali tidak pakai <a href="http://jekyllrb.com/">Jekyll</a> karena instalasi Jekyll yang lama terhapus saat overhoul sistem. Pada saat itu aye merasa sistem sudah terlalu berat dan kemudian diputuskan untuk menghapus semua paket aplikasi ( + libs ) dan memulai ulang memasang paket yang diperlukan saja. Termasuk Jekyll yang saat itu terhapus dan lupa memasangnya kembali.

Ok, Jekyll sudah hadir dengan versi terbaru 1.0.0.2 dengan banyak perubahan dan penambahan fitur. Membuat aye ingin untuk memasang dan memakainya kembali. Maka diputuskan untuk menginstall kembali blog engine ini. Tentu saja sistem aye masih sama yaitu Distro Archlinux, Kernel 3.9.4.1, python3, Ruby 2.2.

Seperti yang sudah ditulis di dokumentasi Jekyll, bahwa untuk memasang Jekyll diperlukan :

<ol>
<li>Ruby</li>
<li>RubyGems</li>
</ol>

Maka perlu terlebih dahulu memasang Ruby dan RubyGems baru kemudian memasang Jekyll dengan perintah sebagai berikut :

<pre>
	<code>
	# pacman -S ruby ruby-docs
	# gem install jekyll
	</code>
</pre>	

Setelah Jekyll terpasang,... ya aye masih punya folder berisi data Jekyll sebelumnya, maka aye perlu masuk ke folder tersebut dan melakukan perintah

<pre>
	<code>
	$ jekyll build
	</code>
</pre>	

untuk membangun ulang direktori blog Jekyll aye. Tapi oh .. ternyata ada masalah!! . Jekyll melaporkan ada masalah pada saat proses build. Disitu disebutkan bahwa Liquid Ext tidak dapat menjalankan proses dengan baik karena <code>pygments</code> tidak bisa dijalankan / ditemukan?!.

<pre>
	<code>
	Liquid Exception: Failed to get header. in post-with-code-snippets.textile
	</code>
</pre>	

Apa ini? ternyata <code>pygments</code> hanya jalan di python2. Masalahnya adalah Archlinux memakai python3 sebagai standarnya. Akhirnya perlu pasang <code>python2</code>.

<pre>
	<code>
	# pacman -S python2
	</code>
</pre>	

Test lagi, dan sukses!! Sukses mendapatkan error yang sama lagi  Kenapa? . Ya, terpaksa harus pasang <code>virtualenv</code> untuk jalankan proses <code>import pygments</i> di python2.

<pre>
	<code>
	# pacman -S python2-virtualenv
	$ mkdir -p ~/.virtualenv/python2
	$ virtualenv2 ~/.virtualenv/python2
	</code>
</pre>	

Dari sini aye masih kena error lagi, kali ini masalah di <code>python2-distutils-extra</code> dilaporkan tidak terpasang / tersedia. Aye sudah pasang <code>python2-distutils-extra</code> versi untuk python2 maupun python3, tapi masih saja <code>virtualenv</code> tidak mau jalan.

Sampai akhirnya bisa jalan dengan melewatkan proses pemasangan <code>setuptools</code> dengan perintah :

<pre>
	<code>
	$ virtualenv2 --nosetuptools ~/.virtualenv/pyhton2
	</code>
</pre>	

setelah sukses, mari kita aktifkan <code>virtualenv</code> :

<pre>
	<code>
	$ source ~/.virtualenv/pyhton2/bin/activate
	(python2) $ cd ~/Github/kuspoes/jekyll build
	</code>
</pre>	

Yes, sukses!!!. Setelah itu keluar dari <code>virtualenv</code> :

<pre>
	<code>
	(python2) $ jekyll build
	(python2) $ jekyll serve
	(python2) $ deactivate
	</code>
</pre>	

Dan insya Allah, Jekyll sudah bisa jalan di <code>localhost:4000</code> . Dan mari siap - siap aktif menulis di blog lagi. Tinggal tulis, dan biarkan Jekyll yang menyelesaikan sisanya. fnd
